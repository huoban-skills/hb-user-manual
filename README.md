# hb-user-manual

伙伴云应用模块使用手册生成 Skill。

根据伙伴云应用的真实配置——表结构、关联关系、自动化、审批流程、表单布局——生成面向业务人员的模块使用手册。读者是一线操作人员，不是产品经理或开发：手册用大白话讲"这个单据怎么填、这个按钮按下去会发生什么、这个数是怎么算出来的"。

## 适用场景

- 使用手册 / 操作手册 / 模块使用说明 / ERP 手册
- 给业务人员看的系统操作指南
- 将伙伴云应用模块整理成可发布的 Markdown 手册，并同步到飞书文档

当用户提到"使用手册""操作手册""模块使用说明""帮我写这个模块怎么用""这个模块的使用指南""ERP手册"等场景时，优先使用本 Skill。

## 核心原则

- **数据驱动，不靠猜**：所有内容来自 CLI/API 采集的真实配置，拿不准的标 `[待确认]`，不编造。
- **关键节点等用户确认**：表范围由用户拍板，手册经用户审阅后才同步飞书。
- **面向业务人员写作**：禁止技术术语，计算逻辑、审批环节、单号规则都翻译成业务语言。

## 工作流程

1. **确定范围**：拉取工作区表清单，用户确认哪些表纳入手册及章节顺序。
2. **批量采集元数据**：对每张表采集——
   - 字段结构（类型、必填、选项值、关联目标表）
   - 自动化清单（按钮、数据触发、校验规则）及节点详情
   - 审批流程（流程中心的流程定义 + 从实例日志还原的审批环节链）
   - 表单布局（主区字段排列顺序、详情页标签页/子表清单）
   - 深层配置（计算字段公式、自动编号规则、默认值、字段显示条件、打印模板、数值单位等）
3. **按模板生成文档**：依据 `references/writing-guide.md` 的写作规范生成 Markdown 手册，待校正处标 `[待确认]` / `[待补充]`。
4. **用户确认后同步飞书**：通过 lark-cli 创建或更新飞书云文档，并复核导入结果。

## 数据采集架构

| 信息 | 采集通道 | 说明 |
|------|----------|------|
| 表清单、字段结构 | `hac table list-tables` / `get-table` | full 模式含计算公式、编号规则、默认值等深层配置 |
| 表单布局 | `hac table form-layout get` | 字段表按表单实际顺序排列，标签页清单含隐藏标记 |
| 审批流程 | `hac procedures list-procedures` 等 | 审批环节链从已完成实例的执行日志还原 |
| 自动化清单（按表枚举） | `scripts/collect_meta.py` | hac 暂无此能力，必须用内置脚本 |
| 单条自动化详情 | `hac automation get` | 已知 automation_id 时使用 |

hac 不可用时，`scripts/collect_meta.py` 可独立覆盖表清单、字段结构和自动化采集（不含流程中心）。

## 依赖

- [hac（huoban-app-cli）](https://www.npmjs.com/package/huoban-app-cli)：主采集通道，需完成认证配置。
- `scripts/` 内置 Python 客户端：按表枚举自动化清单，以及 hac 不可用时的兜底采集，依赖 `HB_*` 环境变量认证。
- [lark-cli](https://www.npmjs.com/package/@larksuite/cli)（可选）：阶段四同步飞书文档时需要。

## 安装

将本仓库克隆或同步到 Claude Skills 目录：

```bash
git clone https://github.com/huoban-skills/hb-user-manual.git ~/.claude/skills/hb-user-manual
```

如果目录已存在，可先确认本地没有未保存修改后再更新。

## 文件

- `SKILL.md`：Skill 主说明文件，包含触发规则、采集命令、流程和执行约束。
- `references/writing-guide.md`：使用手册写作规范（文档结构、字段表写法、审批写法、写作风格）。
- `scripts/collect_meta.py`：元数据采集脚本（字段结构 + 自动化清单 + 节点详情）。
- `scripts/vendor/huoban_automation/`：内置的最小 Huoban API 客户端，供采集脚本独立运行。
- `CHANGELOG.md`：版本变更记录。
